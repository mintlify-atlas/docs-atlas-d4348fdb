## Prerequisites

- An Android project using Jetpack Compose
- Gradle with kapt or KSP configured

## Setup

<Steps>
  <Step title="Add Gradle dependencies">
    Add the three Showkase artifacts to your module's `build.gradle`. If you have a multi-module project, add these dependencies to every module that contains UI elements you want to display.

    The recommended setup limits the UI browser to debug builds only:

    <Tabs>
      <Tab title="KSP">
        ```kotlin build.gradle.kts
        dependencies {
            debugImplementation("com.airbnb.android:showkase:1.0.5")
            implementation("com.airbnb.android:showkase-annotation:1.0.5")
            kspDebug("com.airbnb.android:showkase-processor:1.0.5")
        }
        ```
      </Tab>
      <Tab title="kapt">
        ```kotlin build.gradle.kts
        dependencies {
            debugImplementation("com.airbnb.android:showkase:1.0.5")
            implementation("com.airbnb.android:showkase-annotation:1.0.5")
            kaptDebug("com.airbnb.android:showkase-processor:1.0.5")
        }
        ```
      </Tab>
    </Tabs>

    <Tip>
      `showkase-annotation` uses `implementation` (not `debugImplementation`) so that annotations
      are available in all build variants. The processor and the browser Activity itself are
      debug-only.
    </Tip>
  </Step>

  <Step title="Annotate a composable">
    Add `@ShowkaseComposable` to any zero-parameter `@Composable` function you want to appear in the browser. If you already use `@Preview`, no changes are needed — Showkase picks those up automatically.

    ```kotlin
    import com.airbnb.android.showkase.annotation.ShowkaseComposable
    import androidx.compose.runtime.Composable

    @ShowkaseComposable(name = "Primary button", group = "Buttons")
    @Composable
    fun PrimaryButtonPreview() {
        PrimaryButton(label = "Confirm")
    }
    ```

    Or use the existing `@Preview` annotation you already have:

    ```kotlin
    import androidx.compose.ui.tooling.preview.Preview
    import androidx.compose.runtime.Composable

    @Preview(name = "Primary button", group = "Buttons")
    @Composable
    fun PrimaryButtonPreview() {
        PrimaryButton(label = "Confirm")
    }
    ```

    <Note>
      The annotated function must have no parameters (or use `@PreviewParameter`). If your
      composable requires arguments, wrap it in a zero-parameter preview function.
    </Note>
  </Step>

  <Step title="Create a ShowkaseRootModule">
    In your **root** (app) module, create a class that implements `ShowkaseRootModule` and annotate it with `@ShowkaseRoot`. This tells Showkase which module to treat as the aggregation point for all UI elements across your project.

    ```kotlin
    import com.airbnb.android.showkase.annotation.ShowkaseRoot
    import com.airbnb.android.showkase.annotation.ShowkaseRootModule

    @ShowkaseRoot
    class MyRootModule : ShowkaseRootModule
    ```

    You only need one `@ShowkaseRoot` in your entire project, and it must live in the module that has a dependency on all other modules.
  </Step>

  <Step title="Launch the browser">
    Start the Showkase browser Activity from anywhere in your app — a debug menu, a shake gesture handler, or directly from `onCreate` during development.

    ```kotlin
    import com.airbnb.android.showkase.models.Showkase

    // From an Activity:
    startActivity(Showkase.getBrowserIntent(this))

    // From a Fragment or other context:
    startActivity(Showkase.getBrowserIntent(requireContext()))
    ```

    <Note>
      `getBrowserIntent` is a generated extension function. It becomes available only after you
      build the project at least once following the previous steps. If your IDE shows an
      unresolved reference, run a Gradle build first.
    </Note>
  </Step>
</Steps>

## What you get

After completing setup, the generated browser gives you:

- A searchable list of all annotated components, colors, and typography
- Five automatic permutations per component: basic, dark mode, RTL, font scaled, and display scaled
- Group-based navigation matching the `group` values in your annotations

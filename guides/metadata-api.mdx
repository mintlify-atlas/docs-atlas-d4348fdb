---
title: "Metadata API"
description: "Access structured data about all Showkase elements at runtime using Showkase.getMetadata()."
---

`Showkase.getMetadata()` returns a `ShowkaseElementsMetadata` object containing structured information about every composable, color, and typography entry registered with Showkase. You can use this data to build custom component browsers, drive screenshot tests, run static analysis, or validate your design system in CI.

<Note>
  `getMetadata()` is a generated extension function on the `Showkase` object. You must **build the project at least once** after adding your annotations before this function becomes available.
</Note>

## Basic usage

```kotlin
val metadata = Showkase.getMetadata()

val components = metadata.componentList   // List<ShowkaseBrowserComponent>
val colors     = metadata.colorList       // List<ShowkaseBrowserColor>
val typography = metadata.typographyList  // List<ShowkaseBrowserTypography>
```

## Data classes

### ShowkaseElementsMetadata

The top-level container returned by `getMetadata()`.

<ResponseField name="componentList" type="List<ShowkaseBrowserComponent>" required>
  All composable components registered with Showkase via `@ShowkaseComposable` or `@Preview`.
</ResponseField>

<ResponseField name="colorList" type="List<ShowkaseBrowserColor>" required>
  All colors registered with `@ShowkaseColor`.
</ResponseField>

<ResponseField name="typographyList" type="List<ShowkaseBrowserTypography>" required>
  All text styles registered with `@ShowkaseTypography`.
</ResponseField>

---

### ShowkaseBrowserComponent

Represents a single registered composable component.

<ResponseField name="componentKey" type="String" required>
  A unique key identifying this component entry, generated by Showkase.
</ResponseField>

<ResponseField name="group" type="String" required>
  The group this component belongs to, as specified in the annotation.
</ResponseField>

<ResponseField name="componentName" type="String" required>
  The display name of the component, as specified in the annotation.
</ResponseField>

<ResponseField name="componentKDoc" type="String" required>
  The KDoc comment attached to the annotated function, if present. Empty string if none.
</ResponseField>

<ResponseField name="component" type="@Composable () -> Unit" required>
  The composable lambda that renders this component. Invoke this inside a `setContent` or another composable to render the preview programmatically.
</ResponseField>

<ResponseField name="styleName" type="String?" default="null">
  The style variant label set via `styleName` in the annotation. `null` if the component has no style variants.
</ResponseField>

<ResponseField name="isDefaultStyle" type="Boolean" default="false">
  `true` if this component was annotated with `defaultStyle = true`.
</ResponseField>

<ResponseField name="widthDp" type="Int?" default="null">
  The constrained render width in dp, or `null` if no width constraint was set.
</ResponseField>

<ResponseField name="heightDp" type="Int?" default="null">
  The constrained render height in dp, or `null` if no height constraint was set.
</ResponseField>

<ResponseField name="tags" type="List<String>" default="emptyList()">
  Optional string tags attached via the `tags` annotation property. Useful for filtering in custom browsers.
</ResponseField>

<ResponseField name="extraMetadata" type="List<String>" default="emptyList()">
  Arbitrary string values attached via the `extraMetadata` annotation property. Not used by the standard Showkase browser — available for custom tooling such as attribution or static analysis.
</ResponseField>

<ResponseField name="screenshotConfig" type="ScreenshotConfig" default="ScreenshotConfig.SingleStaticImage">
  Configuration for how screenshot tests should capture this component.
</ResponseField>

---

### ShowkaseBrowserColor

Represents a single registered color.

<ResponseField name="colorGroup" type="String" required>
  The group this color belongs to.
</ResponseField>

<ResponseField name="colorName" type="String" required>
  The display name of the color.
</ResponseField>

<ResponseField name="colorKDoc" type="String" required>
  The KDoc comment attached to the annotated property, if present. Empty string if none.
</ResponseField>

<ResponseField name="color" type="androidx.compose.ui.graphics.Color" required>
  The actual `Color` value.
</ResponseField>

---

### ShowkaseBrowserTypography

Represents a single registered text style.

<ResponseField name="typographyGroup" type="String" required>
  The group this text style belongs to.
</ResponseField>

<ResponseField name="typographyName" type="String" required>
  The display name of the text style.
</ResponseField>

<ResponseField name="typographyKDoc" type="String" required>
  The KDoc comment attached to the annotated property, if present. Empty string if none.
</ResponseField>

<ResponseField name="textStyle" type="androidx.compose.ui.text.TextStyle" required>
  The actual `TextStyle` value.
</ResponseField>

## Use cases

### Custom component browser

Use the metadata to build a fully custom UI for browsing your design system:

```kotlin
val metadata = Showkase.getMetadata()

// Group components by their group name
val groupedComponents = metadata.componentList.groupBy { it.group }

groupedComponents.forEach { (group, components) ->
    println("Group: $group")
    components.forEach { component ->
        println("  - ${component.componentName} (style: ${component.styleName ?: "default"})")
    }
}
```

### Screenshot testing

Iterate over all registered components and render each one in a screenshot test:

```kotlin
@Test
fun screenshotAllComponents() {
    val metadata = Showkase.getMetadata()

    metadata.componentList.forEach { component ->
        composeTestRule.setContent {
            component.component()  // invoke the composable lambda
        }
        // capture screenshot with your preferred tool
        captureScreenshot(name = "${component.group}_${component.componentName}")
    }
}
```

### CI validation

Verify that all expected components are present in your design system before merging:

```kotlin
@Test
fun designSystemContainsRequiredComponents() {
    val metadata = Showkase.getMetadata()
    val componentNames = metadata.componentList.map { it.componentName }

    assertThat(componentNames).contains("Primary Button")
    assertThat(componentNames).contains("Secondary Button")
    assertThat(componentNames).contains("Input Field")
}
```

<Tip>
  The `extraMetadata` field on `ShowkaseBrowserComponent` is designed for exactly this kind of tooling. Attach owner teams, Figma node IDs, or deprecation flags via `extraMetadata` in the annotation, then read them back here for custom analysis.
</Tip>
